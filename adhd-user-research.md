# ADHD 项目 - 用户调研模板

> 沉浸用户 + 跟家长聊天的全流程指南

---

## 一、为什么要做用户调研

### 现实

```
工程师容易犯的错:
  ❌ 自己想需求
  ❌ 闭门造车
  ❌ 假设用户「应该」需要什么

真相:
  ✅ 真实用户的痛点跟你想的不一样
  ✅ 听 20 个家长真实倾诉,价值 > 看 100 篇论文
  ✅ MVP 阶段调研 1 小时,后期能省 100 小时返工
```

### 你的天然优势

**「我自己就是 ADHD 家长」** → 你比任何 PM 更懂这个群体

但**不要只用自己的经验**,因为:
- 你可能是少数(高知 + 工程师)
- 你的孩子情况可能是个例
- 你不代表 2300 万家庭

**用自己的视角入门,用别人的视角验证**。

---

## 二、调研目标

### MVP 阶段要回答的 5 个问题

```
Q1: ADHD 家长每天最痛苦的 3 个时刻是什么?
Q2: 他们现在怎么应对?用什么工具?
Q3: 这些方法/工具的不足是什么?
Q4: 如果有个 App,他们最希望解决哪 1 个问题?
Q5: 他们愿意为这个 App 付多少钱?
```

**这 5 个问题的答案 = 你产品的核心定位**

## 四、跟家长 1V1 聊天的方法

### 找到聊天对象

```
潜伏 2 周后,你会注意到:
  - 哪几个家长经常发言
  - 哪几个家长很真诚
  - 哪几个家长情绪强烈(最痛的)

主动私信他们:
  "我是 XXX 妈妈/爸爸,看到您分享的 XXX,
   特别有共鸣。我想做一个工具,想先听听您
   的真实感受。能聊 30 分钟吗?方便的话语
   音也行。完全免费,不推销任何东西。"

成功率: 30-50%(高于陌生人开发)
```

### 聊天准备

**提前做**:
- 看对方的所有发言,了解情况
- 准备 10 个问题(灵活调整)
- 录音(征求同意)
- 准备一个小礼物答谢(微信红包 20-50 元)

**心态**:
- 你是「**取经者**」,不是「**销售**」
- 多听少说(80% 时间听对方说)
- 不要预设答案
- 不要打断
- 不要替对方总结(让对方完整表达)

---

### 10 个核心问题模板

#### 开场(共情破冰)

```
1. "您家孩子多大?男孩女孩?什么时候发现 ADHD 的?"
2. "诊断那天您是什么感受?"(共情,建立连接)
```

#### 痛点挖掘

```
3. "您觉得带这个孩子最难的是什么?"
4. "请描述一下昨天和孩子相处中最崩溃的一刻?"(细节!)
5. "如果有个魔法能解决一件事,您最希望解决什么?"
```

#### 现状了解

```
6. "您现在用什么方法帮助孩子?"
   (吃药?上课?用 App?)
7. "这些方法效果怎么样?哪里不满意?"
```

#### 需求挖掘

```
8. "如果有个 App,能帮 ADHD 家长解决一个问题,
   您最希望它能解决什么?"
9. "您试过什么 ADHD 相关的 App?为什么没坚持?"
```

#### 付费验证

```
10. "您每年大概花多少钱在孩子 ADHD 上?
    (吃药/训练/咨询)
    如果有个工具真的有用,您每月愿意付多少钱?"
```

---

### 聊完后做的事

**当天**:
- 整理录音 → 转文字
- 提炼 3-5 个关键洞察
- 写到「**用户洞察库**」(后面会建)

**1 周后**:
- 主动反馈:「您之前提到的 XXX,我做了这个 demo,
  您看是不是您说的意思?」
- 把对方变成「**种子用户**」(MVP 上线第一波)

## 六、调研常见误区

### ❌ 误区 1: 问「你需要什么功能?」

**问题**: 用户不知道自己需要什么(乔布斯名言)

**正确做法**: 问场景和痛点
```
❌ "你需要 AI 任务拆分吗?"
✅ "你给孩子布置任务时怎么做?"
```

---

### ❌ 误区 2: 引导对方说 yes

**问题**: 为了验证自己的想法,问引导性问题

**正确做法**: 中立提问
```
❌ "如果有个 App 能自动拆任务,你会用吗?"(对方礼貌说会)
✅ "你现在拆任务有什么困难?"(听真实问题)
```

---

### ❌ 误区 3: 只听好话

**问题**: 对方礼貌客气说「不错」「挺好」

**正确做法**: 追问「真的会付费吗」
```
❌ "你觉得这个功能好吗?" → "挺好的"
✅ "你愿意为这个功能付 99 元吗?"(看真实反应)
```

---

### ❌ 误区 4: 推销自己的方案

**问题**: 聊到一半开始介绍自己的 App

**正确做法**: 第一次聊只听,第二次再展示

---

### ❌ 误区 5: 样本太少

**问题**: 聊了 3 个家长就以为了解市场

**正确做法**: 至少聊 **20 个**,有统计意义

## 七、调研时间表

### Month 1: 沉浸(每天 30 分钟)

```
Week 1: 加 5 个群,潜伏听
Week 2: 关注小红书博主,看评论区
Week 3: 知乎答题 1 篇,被动引流
Week 4: 主动私信 5 个家长,约聊
```

### Month 2: 深度调研(每周 2 次聊天)

```
Week 5-8: 每周聊 2-3 个家长
        累计 8-12 个
        每次聊后整理洞察
```

### Month 3: 数据整理

```
Week 9-10: 把 20 个家长的洞察汇总
         找出共性痛点 (top 5)
         决定 MVP 核心功能 (聚焦 1-2 个)
```

## 八、调研产出的「金矿」

### 1 个月调研能给你:

```
✅ 20 个真实家长的痛点画像
✅ 10 个核心需求 (共识 > 80%)
✅ 5-8 个种子用户(MVP 上线立即用)
✅ 3-5 个潜在 KOL(可帮你传播)
✅ 100+ 篇小红书内容素材(他们的故事)
✅ 1 个产品定位(经过验证)
✅ 1 个付费价格区间(用户能接受)
```

**这些是你创业最珍贵的资产,比 100 行代码值钱**。

## 九、给你的 3 个忠告

### 忠告 1: 不要跳过调研直接写代码

**最常见的死法**:
```
工程师 → 兴奋写代码 → 1 个月做出 MVP → 0 用户用 → 放弃
```

**正确顺序**:
```
1 个月调研 → 3 个月开发 → 100 个用户立即用
(因为你做的是真需求)
```

---

### 忠告 2: 听用户说,但不照搬

**例子**:
- 用户说「我需要 ADHD 知识库」
- 你做了一个 100 篇文章的知识库
- 用户说「太长了不看」

**真相**: 用户要的不是「文章」,是「**快速找到答案**」 → 应该做 AI 问答

**经验**: 用户描述的是「**症状**」,你要看到「**病因**」

---

### 忠告 3: 留下你的影响力

**每次聊完家长**:
- 微信里加好友
- 关注对方小红书
- 真诚帮他/她解决一个具体小问题
- 偶尔分享你的进度

**长期收益**:
- 他们会成为你的种子用户
- 他们会帮你传播
- 他们会给你最真实的反馈

**做创业 = 做长期关系**,不是一次性交易。
